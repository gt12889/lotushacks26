## Codebase Patterns
- Frontend uses Tailwind v4 with `@tailwindcss/postcss` — no tailwind.config file
- Custom theme colors defined via `@theme inline` block in globals.css (not tailwind.config)
- Path alias `@/*` maps to `./src/*` in tsconfig.json
- shadcn v4.1.0 uses `base-nova` style (not `new-york`/`default` — those are removed in v4+)
- shadcn init with `-d -y` flags for non-interactive defaults
- App is always dark-themed — no .dark class toggle needed, :root vars set to abyss theme directly
- Components in src/components/ use mixed import styles: relative (`./Foo`) for siblings, alias (`@/components/Foo`) from app pages — replace both when swapping components
- shadcn Badge v4 uses `useRender` from `@base-ui/react` — pass `className` overrides to customize colors
- shadcn Collapsible v4 uses `defaultOpen` prop and manages state internally — no useState needed for expand/collapse
- 21st.dev registry may return 500 errors — implement ReactBits components manually when needed
- Custom Counter component at ui/counter.tsx provides animated number transitions (easeOutExpo, requestAnimationFrame)
- Custom AnimatedList component at ui/animated-list.tsx wraps keyed children with CSS slide-in animation (plays once per mount)
- Custom GlareHover component at ui/glare-hover.tsx adds mouse-tracking radial gradient glare effect on hover
